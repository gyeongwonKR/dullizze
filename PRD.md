## 1. 제품 개요

### 1.1 한 줄 정의
주제 한 줄 입력 → 정보/지식형 9:16 YouTube 쇼츠를 **고정 템플릿 기반으로 자동 생성·업로드**해 주는 **구독형 웹 서비스**.

### 1.2 목적
- 영상 편집·디자인 기술이 없는 사용자가 **가입 후 클릭 몇 번으로** 쇼츠를 양산.
- 우리가 인프라·API·템플릿을 운영하고 **월 구독료**로 수익.
- 핵심 차별점: **일관된 품질(템플릿) + 오류 없는 산출 + 합리적 원가**.

### 1.3 제품 3대 가치 (이 PRD의 우선순위)
1. **영상 퀄리티 일관성** — 디자인은 사람이 만든 템플릿이 보장. AI는 "내용"만 채운다.
2. **무오류 산출** — 모든 단계 멱등·검증·폴백. AI가 레이아웃/코드를 생성하지 않음.
3. **원가·토큰 효율 + 유지보수성** — 단일 배포, 모델 다운그레이드(Haiku), 캐싱·배치.

## 4. 시스템 아키텍처 (웹 SaaS)

```
[브라우저 (Next.js 등 웹앱)]
   │  로그인 / 채널연결 / 주제·큐 / 템플릿선택 / 생성요청 / 검수 / 공개
   ▼
[API 서버 (FastAPI/Node)]  ── 인증, 과금, 쿼터, 잡 생성
   │
   ├─ [DB (Postgres)]  사용자·채널·잡·템플릿·사용량
   ├─ [오브젝트 스토리지 (S3 등)]  산출물(mp4·assets·script)
   └─ [잡 큐 (Redis/RQ·Celery)]
            │
            ▼
   [렌더 워커 (컨테이너, v1 파이프라인 코어 재사용)]
     ① 주제 → ② 대본(Claude, 캐싱/배치)
     ③ TTS(유료) → ④ 이미지(Flux) → ⑤ 자막타이밍(TTS 경계+그룹핑)
     ⑥ 템플릿 렌더(Remotion+ffmpeg) → ⑦ 메타데이터 → ⑧ 업로드(YouTube OAuth)
            │
            ▼
   [Discord/웹 알림] → [사용자 검수] → 공개
```

- **멀티테넌시**: 모든 데이터·잡은 `user_id`/`channel_id`로 격리.
- **렌더 워커**: stateless 컨테이너. 수평 확장(큐 길이 기반 오토스케일). **서버 렌더 비용이 최대 원가** → 워커 효율이 사업성 핵심.
- **YouTube 연동**: 사용자별 OAuth 토큰 저장(암호화), 토큰 갱신 자동.

| # | 단계 | 입력 | 출력 | v2 변경점 |
|---|---|---|---|---|
| 1 | 주제 pop | 큐/입력 | 주제+컨텍스트 | 멀티유저 큐 |

| 5 | 자막 타이밍 | TTS 단어경계 | `captions.json` | **구문 그룹핑**(§7), Whisper 제거 |

## 6. 템플릿 시스템 (퀄리티·무오류의 핵심)

- **디자인은 코드로 고정**: Remotion 컴포넌트로 만든 **버전화된 템플릿 세트**(레이아웃·폰트·자막 스타일·전환·색감·세이프영역).
- **AI는 디자인을 생성하지 않는다.** AI 출력은 `script.json`(텍스트)와 이미지 프롬프트뿐. 템플릿이 그 데이터를 받아 렌더.
- **카테고리별 템플릿**: 예) 역사/다큐, 잡지식/팝, 뉴스/앵커. 각 템플릿은 `props` 스키마(제목·문장·이미지·자막·BGM)를 고정.
- **품질 게이트**: 렌더 전 props 스키마 검증(zod), 세이프영역·자막 길이·이미지 수 체크 → 실패 시 잡 중단·알림(깨진 영상 방지).
- **유지보수**: 템플릿 추가/수정이 곧 제품 업데이트. AI 프롬프트와 분리되어 회귀 위험 낮음.

## 7. 자막 전략 (구문 단위 — v2 신규)

**문제(v1)**: 어절(띄어쓰기)마다 자막 1개가 표시되어 쇼츠답지 않음.

**해결**:
- **타이밍 소스**: TTS 단어경계.
  - edge-tts: `WordBoundary` 이벤트(offset/duration).
  - 유료 TTS: 단어 타임스탬프 제공 시 사용, **미제공 시 강제정렬 폴백**(대본 텍스트 기준 정렬).
- **구문 그룹핑 규칙**(한 호흡 = 한 줄):
  1. 문장부호(`. , ! ? …`)에서 우선 분할.
  2. 긴 문장은 어절을 누적하되 **한 줄 ≤ 16자 또는 ≤ 4어절**.
  3. **한 줄 ≤ 2.5초**.
  4. 어절 간 **멈춤 ≥ 0.4초**면 그 지점에서 끊기.
- **오류 차단(코드 불변식)**: 어절 중간 분할 금지, 빈 자막 금지, 최대 길이 초과 시 자동 줄바꿈, 한 화면 최대 2줄.
- **표시**: 한 번에 1구문. (선택) 현재 어절 가라오케 하이라이트.
- **산출**: `captions.json` = 구문 배열 `[{text, startMs, endMs, words?}]`.

→ **faster-whisper 제거**: 서버 1~3GB 모델·CPU 시간 제거, 싱크 정확도 향상, 유지보수 단순화.

## 8. AI / 토큰 전략

- **기본 모델: Claude Haiku 4.5** (`claude-haiku-4-5-20251001`). 쇼츠 대본/메타데이터는 Haiku로 충분. 고급 플랜에서만 Sonnet 4.6 옵션. **Opus는 거의 불필요.**
- **프롬프트 캐싱**: 대본 형식 지시(시스템 프롬프트)는 정적 → `cache_control`로 캐싱. 연속/배치 생성 시 입력 토큰 절감.
- **Batch API**: 예약·야간 대량 생성은 Message Batches(**50% 할인**, 비동기). 인터랙티브 단건은 동기+캐싱.
- **중요 사실 — 토큰은 최소 원가**: 편당 대본 비용은 Haiku ~$0.01(Opus 써도 ~$0.12) 수준. **실제 원가는 이미지(유료)와 서버 렌더 컴퓨팅**이다. 토큰 최적화는 위생 수준이고, **원가 관리의 본 레버는 렌더 효율·이미지 비용·쿼터**다.

## 10. 검수 & 품질 게이트 (멀티유저라 더 중요)

- 업로드는 항상 **비공개**로. 공개 전환은 사용자 확인.
- **자동 품질 검사**: 길이 ≤60초, 자막 길이/세이프영역, 이미지 수, 메타데이터 완성, 욕설 필터, **AI 생성 공시 메타 포함**.
- **팩트 리스크**: 정보형 콘텐츠는 사실 오류가 신뢰도에 치명적이며 사용자 수만큼 증폭 → 완전 무인 공개는 제공하지 않음(검수 필수, 룰 기반 보조만).

## 11. 사용 가이드 (온보딩 플로우)

1. **가입/로그인** (이메일·OAuth).
2. **플랜 선택/결제**.
3. **YouTube 채널 연결** (Google OAuth 1회, 토큰 자동 갱신).
4. **채널 설정**: 카테고리·톤·기본 템플릿·보이스 선택.
5. **주제 입력**: 직접 입력 또는 큐 등록(부족 시 AI 추천 → 승인).
6. **생성 요청**: 템플릿 선택 → "생성". 진행 상태 실시간 표시.
7. **검수**: 미리보기(웹 플레이어) + 메타데이터 확인.
8. **공개**: 비공개 업로드된 영상을 검수 후 공개 전환.
9. **대시보드**: 사용량/쿼터, 생성 이력, 실패 사유, 재시도.

> 별도 산출물: 신규 사용자용 **튜토리얼 페이지 + FAQ**(채널 연결 실패, 쿼터, 검수 방법, AI 공시 안내).

## 14. Phase 로드맵 (SaaS 기준 재정의)

> v1 Phase 0~1(환경·단일영상)은 완료. 아래는 v2 기준.

### Phase A — 코어 리팩터 (판매 가능 품질의 기반)
- **A1 자막 구문 그룹핑 + Whisper 제거** (§7). 완료기준: 한 호흡 단위 자막, 어절 분할 0건.
- **A2 템플릿 시스템 도입** (§6). 완료기준: 카테고리 1개 템플릿 + props 스키마 검증.
- **A3 유료 이미지(Flux)·유료 TTS 연동 + 폴백**. 완료기준: 안정적 5장 < 30초, 보이스 교체.
- **A4 모델 Haiku 전환 + 프롬프트 캐싱**. 완료기준: 동일 품질, 토큰/원가 로깅.

### Phase B — 멀티테넌트 백엔드
- 인증·DB·잡 큐·렌더 워커 분리·스토리지. 완료기준: 2계정 동시 생성 격리.

### Phase C — 웹 프론트 + 온보딩
- 가입·결제·채널연결·생성·검수·대시보드. 완료기준: 비개발자가 가이드만 보고 1편 완성.

### Phase D — YouTube 업로드·쿼터·과금 연동
- 사용자 OAuth 업로드, 플랜 쿼터, 사용량/원가 집계. 완료기준: 결제→쿼터→업로드 일주.

### Phase E — 운영·스케일·모니터링
- 오토스케일, 비용 대시보드, 실패 알림·복구, 배치 생성.

## 15. 비용 한도 & 모니터링

- 사용자/플랜별 **월 쿼터**로 원가 상한.
- 전역 일일 예산 가드(`DAILY_BUDGET_USD`)·이상 급증 알림.
- 잡별 원가(토큰·이미지·렌더초) 기록 → 마진 분석.

## 16. 리스크 & 제약

- **OAuth 보안**: 토큰 암호화 저장, 최소 권한 스코프.
